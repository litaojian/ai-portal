# 功能设计规范

## 1. 交互设计 (UX)
- **反馈机制**：
    - **加载中**：异步操作（如提交表单、加载列表）必须显示 Loading 状态（Spinner 或 Skeleton）。
    - **操作成功**：使用 Toast (`sonner.success`) 提示简短信息。
    - **操作失败**：使用 Toast (`sonner.error`) 提示具体错误原因。
    - **破坏性操作**：删除、重置等操作**必须**使用 Dialog 或 Alert Dialog 进行二次确认。
- **表单体验**：
    - **必填标识**：必填字段的 Label 后必须有红色星号 `*`。
    - **实时验证**：输入框失去焦点或提交时进行校验，并在输入框下方显示红色错误文本。
    - **按钮状态**：提交过程中按钮应禁用 (Disabled) 并显示加载图标。

## 2. 界面设计 (UI)
- **语言**：所有面向用户的界面文本使用 **简体中文**。
- **布局模式**：
    - **管理列表页**：顶部筛选/搜索栏 + 中间表格 + 底部状态/分页栏。
    - **表格操作列**：默认**直接显示 2 个常用操作按钮**（如编辑、查看），非悬停显示，其余操作放入 `DropdownMenu` 下拉菜单中，以兼顾操作便捷性与界面整洁。
    - **表单页/抽屉**：清晰的分组（基本信息、高级配置），底部统一的操作按钮区。
- **组件复用**：
    - 必须优先使用 `components/ui` (Shadcn/UI) 中的组件。
    - 保持风格一致性（圆角、阴影、配色）。

## 3. 列表与分页
- **分页**：所有可能超过 20 条数据的列表**必须**实现分页。
    - 标准页大小：10 或 20 条。
    - UI：显示“共 X 条，第 Y/Z 页”以及“上一页/下一页”按钮。
- **空状态**：
    - 列表无数据时，显示友好的空状态插图或文字（“暂无数据”），并提供“新增”引导按钮（如适用）。
    - 搜索无结果时，提示“未找到相关结果”。

## 4. 异常处理
- **403 无权限**：重定向到登录页或显示“无权访问”提示页。
- **404 未找到**：访问不存在的 ID 时，显示“资源不存在”并提供返回列表的按钮。
- **500 系统错误**：捕获并记录错误，向用户显示通用错误提示，避免暴露堆栈信息。
