# 动态菜单功能需求

## 1. 概述
将当前硬编码的侧边栏菜单改为由后台 API 动态获取，支持基于数据库配置菜单结构，为后续的权限控制（不同角色显示不同菜单）奠定基础。

## 2. 数据模型 (Menu)
核心实体 `Menu`：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **id** | String | 主键 (CUID) |
| **title** | String | 菜单标题 |
| **url** | String | 跳转路径 |
| **icon** | String | 图标名称 (Lucide/Tabler icon name) |
| **group** | String | 分组 (main, secondary, clouds, etc.) |
| **order** | Int | 排序权重 |
| **parentId** | String? | 父菜单 ID (用于子菜单) |

## 3. 功能需求
1.  **数据库存储**：创建 `Menu` 表存储菜单信息。
2.  **数据初始化**：通过 Seed 脚本预置现有菜单数据。
3.  **API 接口**：提供 Server Action `getMenus()` 获取结构化菜单数据。
4.  **前端渲染**：
    *   `AppSidebar` 组件改造为接收 props 数据。
    *   建立 `IconMap` 映射，将数据库中的图标字符串转换为组件。

## 4. 影响范围
*   `prisma/schema.prisma`
*   `components/app-sidebar.tsx`
*   `app/dashboard/page.tsx`
*   `app/projects/page.tsx`
